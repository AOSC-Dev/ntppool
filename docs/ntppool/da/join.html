Hvordan tilmelder jeg mig pool.ntp.org?

Først: Tak for din interesse. Brugen af 'pool'en' er steget enormt og den eneste måde vi kan gøre det nemt for alle at have en server kørende, er ved også at øge antallet af tilmedte servere.

Din server skal have en statisk IP adresse og fast internet forbindelse. Det er meget vigtigt at din IP adresse ikke skifter eller at det gør det meget sjældent (ca 1 gang om året eller mindre). Den krævede båndbredde er relativ lille. Hver klient vil kun sende et par UDB pakker ca hver anden minut eller 20. minut.

Pt. får de fleste servere omkring 5-15 NTP pakker hver sekund, med spidser på 60-120 pakker i sekundet, et par gange om dagen. Dette svare groft til 10-15 Kbit/s med spidser på 50-120 Kbit/s. Projektet vil stødt kræve flere servere, så belastningen skulle ikke gerne stige dramatisk for den enkelte server. Sagt lige ud vil du få brug for en 384-512 Kbit forbindelse (begge retninger).

Here are a few servers with traffic/load graphs: [% i = 0; FOR server = combust.servers_with_urls; %] [% i=i+1; i %] [% END %]

Tilmeldte servere bør ikke have pool.ntp.org som deres upstream server(e), men bør bruge nogen gode servere, disse kan dog sagtens være fra pool'en. Poienten er at disse vælges statisk, frem for at blive valgt tilfældigt ved hver server start. Dette hjælper til at give et acceptabelt service niveau. Note that it is not required that your server is a stratum 1 or 2 server - as this project is about load distribution mostly, there is no reason why a stratum 3 or even stratum 4 server shouldn't join.

We have a page with configuration recommendations for servers joining the pool.

Finally, I must emphasize that joining the pool is a long term commitment. We are happy to take you out of the pool again if your circumstances change, but because of how the ntp clients operate it will take weeks, months or even YEARS before the traffic completely goes away.

If this is all ok, login to the server management page and request your server to be added. If you have any trouble with the system, email me at ask@develooper.com.

I'd ask you to also subscribe to the weblog feed and maybe to the pool mailing list.

It would be nice (but it's not necessary) if you could redirect web requests on port 80 of your time server to the official web page of the project at http://www.pool.ntp.org. If you run Apache, you could do this with:

[% PROCESS tpl/join/virtual_host_example.html %]

Again, this is only if you run a webserver anyway. The official project web page will always be given with the 'www' at the beginning - but sometimes people are lazy and just type pool.ntp.org and are then surprised to just get a random web page.

Once added to the pool the server is constantly monitored for availability and precision. You can watch your server's performance using the web frontend or the management page.
